## Git 관리 규칙

해당 Repository의 Git은 다음과 같이 관리합니다.

<br/>

#### 1. branch 관리

- branch는 다음과 같은 계층 구조로 관리합니다.

  ```
  main
  └ develop
    └ front-end
      └ feature/foo
        ...
    └ back-end
      └ feature/bar
        ...
  ```

<br/>

<br/>

#### 2. commit message

```shell
git commit -m '[월일] | [개발자 이름] | [커밋 내용]'

`ex)` 
git commit -m '0108 | Sinbad | 로그인 기능 구현'
```

- **커밋 내용은 짧은 구문**으로 작성하기!
